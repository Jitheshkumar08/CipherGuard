How It Works

Encryption pipeline
1
AES-256-CBC
Raw image binary encrypted with a 256-bit key and random 16-byte IV in CBC mode. Fastest layer — handles the bulk data.
→
2
Triple-DES-CBC
AES output re-encrypted with a 24-byte 3DES key and 8-byte IV. Independently keyed — a second complete encryption layer.
→
3
RSA-2048-OAEP
Both symmetric keys packed into 80 bytes and encrypted with RSA-2048. The private key never leaves the server.
→
📄
.mlenc file
Binary format: magic header + RSA-encrypted keys + original filename + double-encrypted ciphertext.
.mlenc file format
0–7MLENC0018-byte ASCII magic header
8–11key_lenuint32 big-endian — RSA block size
12–267RSA-encrypted keys256 bytes — AES+3DES keys wrapped by RSA
268–269name_lenuint16 big-endian — filename length
270–NfilenameUTF-8 original filename
N+1–EOFciphertextAES(3DES(image)) — the double-encrypted data
