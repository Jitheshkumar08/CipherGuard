Encryption Pipeline

Three layers. Zero compromise.

Every file travels through an irreversible chain of cryptographic operations before it ever touches storage.

01
AES-256-CBC

Symmetric Core

Your image bytes are encrypted with AES-256 in CBC mode using a random key and IV generated fresh per file.

→
02
3DES-CBC

Double Wrap

The AES ciphertext is encrypted again with Triple-DES CBC — a second independent cipher layer adds serious depth.

→
03
RSA-2048-OAEP

Asymmetric Seal

All symmetric keys are RSA-encrypted with your personal 2048-bit public key. Only your private key can unwrap them.

→
04
.mlenc Container

Signed Package

Everything is packed into a binary .mlenc container with a magic header for tamper detection.

Features

Everything you need to stay secure

From signup to encrypted vault — every step is hardened.

🔑

Per-User Key Vault

At signup, a unique DEK and RSA-2048 key pair are generated just for you. Your private key is encrypted and stored — never raw.

🛡️

Password-Derived KEK

Your password derives a Key Encryption Key via bcrypt. Even if the database is breached, your keys remain protected.

🔒

JWT Authentication

Every request is verified with signed JSON Web Tokens. Sessions expire automatically and logout is clean.

📦

Encrypted File Store

Encrypted .mlenc files are persisted server-side. List, download, decrypt, or delete them any time.

🔄

Safe Password Change

Changing your password re-encrypts the DEK automatically. Existing files stay fully readable — no migration needed.

🎲

Fresh IVs Every Time

Each encryption uses randomly generated IVs, so the same image always produces a different ciphertext. No patterns, ever.

How It Works

From image to encrypted vault in seconds

Simple for users. Complex under the hood.

1

Create Your Vault

Sign up with email and password. CipherGuard instantly generates your unique RSA key pair and encrypts it with your password-derived key.

2

Upload an Image

Drop any JPG, PNG, GIF, WEBP, BMP or TIFF up to 20 MB on the Encrypt page. Your file never leaves your session unprotected.

3

Triple Encryption Runs

AES-256 → 3DES → RSA-2048 all execute server-side using your unlocked keys. A .mlenc file is returned instantly.
